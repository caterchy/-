## 决策记录

### 2026-04-30: 初始项目架构 — Vue 3 + TypeScript + Vite
- **决策**: 使用 Vue 3 Composition API + TypeScript + Vite 构建六爻排盘工具
- **原因**: 类型安全、构建速度快、组合式 API 适合复杂逻辑组织
- **影响**: 项目技术栈定稿，后续所有组件和逻辑基于此架构

### 2026-04-30: 状态管理选型 — Pinia
- **决策**: 使用 Pinia 替代 Vuex 进行状态管理
- **原因**: Pinia 是 Vue 3 官方推荐，TypeScript 支持更好，API 更简洁
- **影响**: 起卦历史、排盘结果等全局状态通过 Pinia store 管理

### 2026-04-30: 持久化方案 — localStorage
- **决策**: 使用 localStorage 存储起卦历史和备注
- **原因**: 纯前端项目，无后端服务，localStorage 足够满足 100 条历史记录的存储需求
- **影响**: 最多存储 100 条记录，超出时自动淘汰最旧记录；浏览器清缓存会导致数据丢失

### 2026-04-30: 多角色 Agent 协作系统
- **决策**: 建立总负责人 + 子代理的多角色协作体系
- **原因**: 不同类型的任务（开发/Debug/测试/管理）需要不同的专业视角
- **影响**: 新增 5 个子 Agent 定义，CLAUDE.md 重构，新增 rules/ 规则目录

### 2026-05-01: 汉字起卦笔画计算 — 接受哈希估算 (B1)
- **决策**: 保留当前 `getStrokeCount` 的哈希公式 `(code - 0x4E00) * 13 + 7) % 30 + 1`，不引入真实笔画数查询表，标记 B1 为 resolved-by-design
- **原因**: 起卦工具的核心需求是确定性地数字映射到上下卦和动爻，哈希分布均匀满足此需求；真实笔画表存在覆盖不全、体积大、语义不一致等问题
- **影响**: B1 关闭，CharacterDivination.vue 保持不变；UI 中保留"非实际笔画数"说明文字

### 2026-05-01: 5.2 UI/布局迭代 — 16 项需求全部完成
- **决策**: 实施 cater/liuyao.md 中 5.2 新要求 + 全局左右对称 + 折叠改为文字链接，共 16 项变更
- **原因**: 用户反馈 UI 布局混乱、排版不清晰、交互不统一，要求全面重构
- **影响范围**:
  - 核心变更: TraditionalView 本变卦左右分栏、全局布局对称化
  - 起卦界面: 备注迁移、动爻○×、摇币三枚独立+统计、数字拆单双数
  - 排盘信息: 伏神按需生成、变卦去六神、排盘去神煞、卦辞选项删除
  - 时间模块: 八字格式规范、旬空/卦身独立、神煞可折叠
  - 交互: 显示开关精简、折叠控件改为文字链接
- **Bug 修复**: TraditionalView 中纳甲详解被变卦条件阻塞无法展开（已修复）
- **构建状态**: npm run build 零错误通过

### 2026-05-01: Bug 修复强制同步机制
- **决策**: Debug/开发修复 Bug 后必须通知项目经理立即更新状态文档，禁止同一 Bug 重复修复
- **原因**: 修复后状态未同步到文档导致总负责人不知情、Bug 被反复分配重复修复
- **影响**: 修改 `agent_hierarchy.md`（新增第5条协作规则）、`feedback_mechanism.md`（新增 Bug 修复闭环流程小节）；涉及 Debug、开发、项目经理三个角色的协作流程变更
